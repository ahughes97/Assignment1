XML Syntax Rules


Six important rules that must be adhered to in order to create a valid XML file:

Rule 1: Closing Tags.
In XML, it is illegal to leave out the closing tag. All elements MUST have a closing tag.

Rule 2: Case Sensitivity.
XML tags are case sensitive therefore the tag < Letter > is different from the tag < letter >. Opening and closing tags must be written using the same case.

Rule 3: Properly Nested Elements.
All elements in XML MUST be properly nested within each other. Therefore if the < i > element is opened inside the < b > element, it must be closed within the < b > element.

Rule 4: Root Elements.
All XML documents must contain one root element that is the parent of all the other elements. Therefore, within the < root > tags are < child > and < subchild > tags.

Rule 5: Quoted Attributed Values.
Elements can have attributes in name and value pairs, similarly to HTML. In XML, the atrribute values must be quoted. For example: < date = "06/10/1997" >.

Rule 6: Comments in XML.
The syntax used for writing comments in XML is similar to the syntax used in HTML. However, two dashes within the comments isn't allowed. Therefore, < !--This is a valid comment-- > but < !--This is an invalid -- comment-- >
